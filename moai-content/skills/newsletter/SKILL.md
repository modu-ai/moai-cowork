# 뉴스레터 (Newsletter)

> moai-content | 뉴스레터 전문 스킬

## 지원 영역

- 뉴스레터 기획부터 발행까지 전 과정
- 구독자 확보 전략 (리드 마그넷, 구독 폼 최적화)
- 오픈율·클릭률 최적화 (제목, 발송 시간, CTA)
- 위클리·먼슬리 뉴스레터 콘텐츠 작성

참조 가이드: `references/newsletter-engine.md`

## 트리거 키워드

뉴스레터, 이메일 뉴스레터, 구독자, 뉴스레터 발행, 위클리, 먼슬리, 콘텐츠 구독

## 사용 예시

- "스타트업 위클리 뉴스레터 첫 호 써줘"
- "이메일 제목 후보 10가지 만들어줘 (오픈율 높이기)"
- "구독자 1,000명 확보 전략 기획해줘"
- "먼슬리 뉴스레터 콘텐츠 캘린더 만들어줘"
- "업계 트렌드 뉴스레터 구성 짜줘"

## 독립 실행 워크플로우

참조 가이드(`references/`)를 사용할 수 없는 경우 다음 단계로 실행합니다.

### 1단계: 뉴스레터 목적 파악
- 목적: 브랜드 인지 / 리드 육성 / 고객 관계 / 수익화
- 발송 주기: 주간 / 격주 / 월간
- 타겟 독자: 직군, 관심사, 구독 동기

### 2단계: 뉴스레터 구조 설계
```
[제목 (Subject Line)]
- 문자 수: 40자 이내 (모바일 잘림 방지)
- 이모지: 1개 이내 (너무 많으면 스팸 처리)
- 개인화: "[이름]님," 접두어 (오픈율 +20%)
- 공식: [숫자/질문/혜택] + [키워드]
  예: "이번 주 읽어야 할 AI 뉴스 3가지"

[프리헤더 (Preview Text)]
- 40~100자 이내 (제목 보완 설명)

[본문 구조]
① 인트로 (3~5문장): 이번 호 핵심 메시지
② 메인 콘텐츠 (1~3개 섹션):
   - 큐레이션: 업계 뉴스 + 편집자 코멘트
   - 인사이트: 독자가 얻을 수 있는 핵심 관점
   - 실용 정보: 도구, 팁, 체크리스트
③ CTA (1개만): 명확한 다음 행동 유도
④ 마무리: 다음 호 예고 + 구독 유지 동기
```

### 3단계: 오픈율 최적화
```
높은 오픈율 제목 공식:
- 질문형: "왜 ~가 실패하는가?"
- 숫자형: "이번 주 놓치면 안 될 5가지"
- 개인화: "[독자가 관심 있는 주제] 완벽 정리"
- 긴박감: "오늘만 공개하는 ~"
- 호기심: "아무도 말하지 않는 ~의 진실"

피해야 할 제목:
- "안녕하세요, [회사명] 뉴스레터입니다"
- "이번 달 소식을 전해드립니다"
- 과도한 이모지 (🔥💯🎁🚀 남발)
```

### 4단계: 구독자 확보 전략
```
리드 마그넷 (구독 유인):
- 무료 템플릿, 체크리스트, 가이드 PDF
- 독점 정보, 할인 코드
- 첫 호 특별 콘텐츠 예고

구독 폼 최적화:
- 입력 필드 최소화 (이메일 1개 → 가장 높은 전환율)
- "스팸 안 보냄" 명시
- 발송 주기와 내용 예시 공개

배포 채널:
- 웹사이트 팝업 / 블로그 하단 / SNS 프로필 링크
- 파트너십: 유사 뉴스레터 상호 교차 홍보
```

### 5단계: 발송 플랫폼 추천
```
국내/글로벌 이메일 발송 도구:
- Stibee (스티비): 국내 최적화, 한국어 지원, 소규모 무료
- Mailchimp: 글로벌 표준, 구독자 500명 무료
- ConvertKit: 크리에이터 특화, 자동화 강점
- Substack: 유료 구독 모델, 빌트인 결제
- Ghost: 블로그+뉴스레터 통합, 자체 운영
```

### 6단계: 콘텐츠 캘린더 설계 (월간)
```
월간 뉴스레터 캘린더 예시 (주간 발행):

1주: 업계 트렌드 요약 (큐레이션 중심)
2주: 심층 인사이트 (오리지널 콘텐츠)
3주: 실용 도구·팁 (How-to 중심)
4주: 독자 사례·Q&A + 다음 달 예고
```

## 실행 규칙

1. 사용자 요청 수신 → 뉴스레터 유형 판별
2. `references/newsletter-engine.md` 존재 시 로드 → 가이드에 따라 실행
3. `--deepthink` 또는 복잡 작업 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 결과물 생성 후 사용자 검토 요청

⚠️ **AI 생성 콘텐츠 주의**: AI가 생성한 뉴스레터 내용은 사실 확인 후 발송하세요. 업계 통계, 링크, 인물 정보는 반드시 원출처를 검증하세요.

## 문제 해결

- **플랫폼 규격 변경**: 이메일 클라이언트(Gmail, 네이버 메일)의 렌더링 방식은 수시로 변경됩니다. 발송 전 주요 클라이언트에서 테스트하세요.
- **브랜드 가이드 미제공**: 브랜드 톤앤매너, 색상, 로고 사용 지침을 제공하면 더 일관된 뉴스레터 템플릿을 설계할 수 있습니다.
- **이미지 생성 실패**: 이 스킬은 텍스트 콘텐츠를 생성합니다. 헤더 이미지는 Canva, Adobe Express 등으로 직접 제작하세요.
- **스팸 필터 걸림**: 과도한 이모지, "무료", "할인" 등 단어 남발은 스팸으로 분류될 수 있습니다. 제목과 본문에서 스팸 유발 단어를 최소화하세요.

## 이 스킬을 사용하지 말아야 할 때

- **블로그 포스팅**: SEO 블로그 글 작성은 `blog` 스킬이 더 적합합니다.
- **SNS 포스팅**: 소셜미디어 콘텐츠는 `social-media` 스킬을 사용하세요.
- **랜딩 페이지**: 구독 유도 랜딩 페이지 전체 설계는 `landing-page` 스킬이 더 적합합니다.
- **광고 카피**: 이메일 외 채널 광고 문구는 `copywriting` 스킬을 사용하세요.
