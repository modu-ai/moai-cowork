# 블로그 (Blog)

> moai-content v1.0.0

## 개요

6개 주요 블로그 플랫폼에 최적화된 블로그 포스팅을 작성한다.
2026년 최신 네이버 C-Rank·D.I.A.+ 알고리즘, GEO(생성형 AI 검색 최적화), 카피라이팅 안티패턴을 반영하여 검색 상위 노출과 독자 전환율 모두를 극대화한다.

⚠️ **AI 생성 콘텐츠 주의**: AI가 생성한 블로그 내용은 사실 확인 후 게시하세요. 통계, 날짜, 인물 정보는 반드시 원출처를 검증하세요.

## 워크플로우

### 1단계: 플랫폼 및 목적 확인
- 발행 플랫폼 선택 (1개 이상)
- 작성 목적:
  - 검색 유입 (SEO/GEO 최적화)
  - 브랜드 인지도 (전문성 확보)
  - 전환 유도 (리드 수집, 상품 구매)
  - AI 검색 인용 (GEO 최적화)
- 타겟 독자 정의

### 2단계: 키워드 및 주제 전략

**네이버 C-Rank 키워드 전략 (2026):**
- 단일 주제 집중: 31개 카테고리 중 하나에 전문화
- 경쟁도 분석: 네이버 키워드 도구로 월간 검색량·경쟁률 확인
- 로얄 키워드 회피: 검색량 대비 문서 수 비율이 낮은 키워드 타겟
- 롱테일 키워드: "스타트업 B2B SaaS 영업 전략" 등 구체적 표현

**D.I.A.+ 검색 의도 분석:**
- 정보형 쿼리: "X란 무엇인가", "X 하는 방법" → 가이드·설명 형식
- 비교형 쿼리: "X vs Y 차이" → 비교 표·체크리스트 형식
- 후기형 쿼리: "X 사용 후기", "X 리뷰" → 경험 기반 서술 필수

**GEO 최적화 키워드 전략 (2026 신규):**
- 질문형 헤딩: "Q: ~는 어떻게 하나요? A: ..." 형식
- 정의형 단락: "~란, ... 이다" 명확한 정의 문단
- AI 인용 가능한 통계·수치 포함

### 3단계: 포스팅 구조 작성

**표준 블로그 포스팅 구조:**
```
[제목] (40자 이내, 핵심 키워드 포함, 호기심 유발)

[도입부] (150~200자)
- 독자의 문제 공감 또는 핵심 인사이트 선 제시
- "이 글을 읽으면 ~를 알 수 있습니다" 명확한 가치 제안

[목차] (네이버·워드프레스 필수)

[본문] (각 섹션 H2/H3 소제목)
- 섹션당 300~500자
- 실제 경험·사례·데이터 포함 (D.I.A. 경험 정보)
- 표·리스트·이미지로 시각화

[결론] (100~200자)
- 핵심 정리 3가지
- CTA (다음 액션 유도)

[참고 자료] (GEO 신뢰도 강화)
```

**플랫폼별 최적 분량:**
| 플랫폼 | 권장 분량 | 이미지 수 |
|--------|-----------|-----------|
| 네이버 블로그 | 1,500~3,000자 | 5~10장 |
| 티스토리 | 2,000~4,000자 | 3~7장 |
| 브런치 | 800~2,000자 | 1~3장 |
| WordPress | 1,500~3,000자 | 3~5장 |
| Ghost | 1,200~2,500자 | 2~5장 |
| Medium | 1,000~2,500자 | 2~4장 |

### 4단계: 네이버 블로그 특화 최적화

**C-Rank 향상 전략:**
- 주제 전문성: 같은 카테고리 글 꾸준히 발행 (주 2~3회)
- 체류 시간: 읽기 시간 5분 이상의 충분한 분량
- 상호작용 유도: 댓글 질문, 이웃 추가 CTA

**D.I.A. 고품질 요소:**
- 실제 경험 기반 후기 포함 (본인 사진, 영수증, 스크린샷)
- 최신 정보 반영 (날짜 명시)
- 이미지에 텍스트 설명 추가 (대체 텍스트)
- AI 생성 텍스트 그대로 사용 금지 (패널티 위험)

**2026년 네이버 최신 변화:**
- 스마트블록 통합: VIEW 탭 → 스마트블록으로 완전 전환
- 영상 콘텐츠 우대: 네이버 클립 첨부 시 추가 노출 혜택
- 지역 콘텐츠 우대: 로컬 키워드 포함 시 지역 검색 노출

### 5단계: 카피라이팅 안티패턴 (2026 회피 목록)

**피해야 할 표현 패턴:**
- "안녕하세요! 오늘은 ~에 대해 알아보겠습니다" → 즉시 핵심으로 시작
- 과도한 이모지 남발 (🔥💯✨) → 1~2개 이내로 제한
- 근거 없는 최상급: "최고", "최대", "No.1" → 수치·출처로 대체
- 클릭베이트 제목: "이것 모르면 손해" → 구체적 혜택 제목으로
- AI 티 나는 문장: "이 글에서는... 살펴보겠습니다. 첫째로..." → 자연스러운 구어체
- 불필요한 반복: 같은 내용을 다른 말로 3번 이상 → 압축
- 과도한 줄바꿈 (한 문장씩 줄바꿈) → 문단 단위로 구성

**효과적인 블로그 카피 원칙:**
- 독자의 '언어'로 쓰기 (전문 용어 최소화, 풀어서 설명)
- 숫자로 신뢰성 강화: "3가지", "5분 만에", "30% 절감"
- 구체적 사례와 결과 제시
- 행동을 유도하는 명확한 CTA

### 6단계: GEO 최적화 적용 (AI 검색 대응)

**AI 검색 인용 최적화:**
- FAQ 섹션 추가: "Q: 자주 묻는 질문, A: 명확한 답변"
- 정의 단락: "~란, [간결한 정의]이다. [추가 설명]..."
- Schema 마크업 권고 (WordPress/Ghost): Article, HowTo, FAQ
- 출처·참고문헌 섹션으로 신뢰도 강화
- robots.txt에서 GPTBot·ClaudeBot·PerplexityBot 차단 금지 확인

## 발행 연동 (선택)

WordPress Cowork 커넥터가 연결되어 있으면 발행까지 자동화합니다.

실행 시 커넥터 연결 여부를 확인합니다:
- **연결됨** → 카피 생성 후 WordPress에 직접 발행 (카테고리, 태그, 특성 이미지 자동 설정)
- **미연결** → "WordPress 커넥터가 연결되어 있지 않습니다. Settings > Connectors에서 연결하세요." 안내 후, 마크다운 카피만 생성

네이버/티스토리/브런치/Ghost 등: 플랫폼에 최적화된 마크다운 카피를 생성합니다 (발행은 수동).

커넥터 설정 상세: `CONNECTORS.md` 참조

## 문제 해결

- **플랫폼 규격 변경**: 네이버·티스토리 알고리즘은 수시로 변경됩니다. 최신 정보는 각 플랫폼 공식 블로그나 SEO 커뮤니티에서 확인하세요. AI가 제공하는 알고리즘 정보는 2026년 기준입니다.
- **브랜드 가이드 미제공**: 브랜드 톤앤매너, 타겟 독자, 금지어가 있다면 미리 알려주세요. 없으면 일반 독자 친화적 문체로 작성합니다.
- **이미지 생성 실패**: 이 스킬은 텍스트 콘텐츠를 생성합니다. 이미지가 필요한 경우 Midjourney, Adobe Firefly 등 이미지 생성 도구를 별도로 활용하거나 `card-news` 스킬을 사용하세요.
- **사실 정보 오류**: AI가 생성한 통계, 날짜, 기업 정보는 반드시 원출처를 확인하세요. 오류 발견 시 수정 요청해 주세요.

## 이 스킬을 사용하지 말아야 할 때

- **인스타그램·SNS 콘텐츠**: 소셜미디어 포스팅은 `social-media` 스킬이 더 적합합니다.
- **카드뉴스·슬라이드**: 비주얼 중심 캐러셀 콘텐츠는 `card-news` 스킬을 사용하세요.
- **이메일 뉴스레터**: 구독자 대상 뉴스레터는 `newsletter` 스킬을 사용하세요.
- **광고 카피·헤드라인**: 전환 최적화 마케팅 문구는 `copywriting` 스킬이 더 적합합니다.
