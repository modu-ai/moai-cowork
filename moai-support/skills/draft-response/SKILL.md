# 응답 초안 작성 (Draft Response)

> moai-support v1.0.0

## 개요

고객 문의에 대한 응답 초안을 한국어 경어로 작성합니다. 이메일, 카카오채팅, 앱 내 메시지, 공식 답변서 등 채널별 어조와 형식에 맞춰 초안을 생성합니다. 2026년 한국 CX 트렌드에 맞춰 공감 우선 + 해결책 명확화 + AI 투명성 원칙을 준수합니다.

## 트리거 키워드

응답 초안, 답변 작성, 이메일 답장, 고객 응대, 메시지 초안, 공식 답변, CS 답변, 답변 템플릿, 문의 답변

## 워크플로우

1. **문의 유형 파악**: ticket-triage 스킬 연동 또는 직접 입력된 문의 내용 분석
2. **채널 선택**: 이메일 / 카카오채널 / 앱 내 채팅 / 공식 서한 / SNS DM
3. **어조 설정**: 
   - 공식: 공문서·법적 답변, 높임말 최고 수준
   - 표준: 이메일·헬프데스크, 정중한 비즈니스 어조
   - 친근: 채팅·SNS, 부드러운 고객 접점 어조
4. **초안 생성**: 공감 → 상황 확인 → 해결책 → 다음 단계 구조로 작성
5. **품질 체크**: 경어 일관성, 개인정보 마스킹, 부정적 표현 검토

## 응답 구조 원칙

```
1. 공감 인사: "고객님, 불편을 드려 진심으로 죄송합니다."
2. 상황 확인: "말씀하신 [주문번호/문제]를 확인하였습니다."
3. 해결책: 구체적인 처리 방안 또는 타임라인 명시
4. 다음 단계: 고객이 해야 할 행동 또는 처리 완료 예정일
5. 마무리: 지속적인 지원 의사 표명
```

## 채널별 어조 가이드

| 채널 | 어조 | 길이 | 특징 |
|------|------|------|------|
| 이메일 | 정중·공식 | 200~400자 | 제목 포함, 서명 필수 |
| 카카오채널 | 친근·간결 | 100~200자 | 이모지 선택적, 빠른 응답 |
| 앱 내 채팅 | 표준·신속 | 50~150자 | 실시간성 강조 |
| 공식 서한 | 격식·법적 | 500자+ | 날짜·수신인·발신인 필수 |
| SNS DM | 친근·공개 | 100자 이내 | 공개 가능성 고려 |

## 2026년 한국 CS 응답 핵심 원칙

- **반복 설명 방지**: 고객 74%가 여러 상담사에게 같은 내용 반복 시 불만 표출. 이전 대화 컨텍스트를 응답에 반영
- **24시간 응대**: 소비자 74%가 24시간 응대 기대. 즉시 응답 불가 시 자동 접수 확인 메시지 초안 포함
- **AI 투명성**: AI 생성 응답임을 표시할 경우 근거 설명 포함 권장 (고객 95% 요구)
- **감정 우선**: 필코노미 트렌드 — 해결책 전 공감 표현이 고객 만족도를 결정

## 산출물

- 채널별 맞춤 응답 초안 (한국어 경어)
- 대안 어조 버전 (공식/표준/친근 3종)
- 후속 조치 안내문
- 자동 접수 확인 메시지 (즉시 응답용)

## 참고 사항

- **경어 레벨**: "합니다/습니다" 체 사용 필수. "해요" 체는 채팅 채널에만 선택적 허용
- **부정 표현 지양**: "안 됩니다", "불가능합니다" 대신 "처리해 드릴 수 있는 방법은..." 형식 사용
- **약속 주의**: 정확한 처리 기한 미확인 시 "영업일 기준 3~5일 내" 등 범위 표현 사용
- **개인정보**: 응답에 고객 주민번호, 카드번호, 계좌번호 직접 기재 금지
