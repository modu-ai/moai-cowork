### 기본 실행 워크플로우 (references 미로드 시)

1. 사용자 요청에서 목표/산출물/대상 추출
2. 해당 도메인의 표준 프레임워크 적용 (예: SWOT, BMC, Porter's 5F, OKR)
3. 초안 생성 → 사용자 검토 요청
4. 피드백 반영하여 최종본 완성

## 트리거 키워드

사업계획서, 창업, 스타트업, 린 캔버스, MVP, 비즈니스 모델, SWOT, OKR, Blue Ocean, 전략, 시나리오, 신시장 진출, 해외 진출

## 워크플로우

### 1단계: 요청 유형 파악
- 목표 산출물 확인 (사업계획서 / 전략 프레임워크 / 비즈니스 모델 / 시나리오)
- 비즈니스 단계 확인 (초기 아이디어 / 성장 단계 / 신시장 진출)
- 사용 가능한 자료 및 데이터 확인

### 2단계: 프레임워크 선택 및 적용
- **스타트업/창업**: 린 캔버스 → 핵심 가정 검증 → MVP 정의
- **비즈니스 모델**: BMC 9블록 → 가치 제안 → 수익 모델
- **전략 분석**: SWOT → SO/WO/ST/WT 전략 도출
- **신시장 진출**: 시장 매력도 + 진입 장벽 + 파트너 전략

### 3단계: 산출물 생성
- 구조화된 전략 문서 초안 작성
- 핵심 가정과 검증 방법 명시
- 리스크 요소 식별 및 대응 방안 포함

### 4단계: 검토 및 정제
- 사용자 피드백 수렴
- 수치/가정 구체화
- 최종 문서 완성

## 사용 예시

**예시 1**: "온라인 영어 교육 스타트업 사업계획서 써줘"
→ 시장 분석 → 린 캔버스 → 경쟁 포지셔닝 → 수익 모델 → 투자 요약 순서로 생성

**예시 2**: "우리 회사 SWOT 분석하고 전략 방향 잡아줘. 자료는 이거야: [파일]"
→ 제공 자료 분석 → SWOT 매트릭스 → SO/WO/ST/WT 전략 → 우선순위 액션플랜

## 실행 규칙

1. 사용자 요청 수신 → 해당 하네스 판별
2. `references/{id}.md` 로드 → 전략 가이드에 따라 실행
3. `--deepthink` 또는 복잡 전략 판단 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 결과물 생성 후 사용자 검토 요청

## MCP 도구 활용 (DART)

DART MCP 서버가 연결되어 있으면 금감원 전자공시시스템을 활용합니다:
- **기업 검색**: 회사명 또는 종목코드로 기업코드 조회
- **공시 조회**: 사업보고서, 분기보고서, 주요사항보고서 등
- **재무 데이터**: 재무제표, 배당 정보, 주요 경영지표
- **활용 순서**: 경쟁사/대상 기업 검색 → 최신 사업보고서 조회 → 재무 데이터 추출 → 전략 분석에 반영

MCP 미연결 시: 사용자 제공 데이터 또는 공개 정보 기반으로 진행.

## 문제 해결

| 상황 | 대응 |
|------|------|
| 참고 자료가 부족한 경우 | 핵심 가정을 명시하고 업계 평균 데이터 기반으로 진행. 사용자에게 추가 자료 요청 |
| 업종이 생소하거나 데이터가 없는 경우 | 유사 업종 사례 참조. 한계를 명시하고 진행 |
| 요청 범위가 너무 넓은 경우 | 우선순위 질문 2~3개로 핵심 범위를 좁힌 후 진행 |
| DART MCP 연결 실패 | 사용자에게 자료 직접 제공 요청. 공개 데이터 기반 진행 |
