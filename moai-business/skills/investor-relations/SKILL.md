### 기본 실행 워크플로우 (references 미로드 시)

1. 사용자 요청에서 목표/산출물/대상 추출 (IR 덱 / 재무 모델 / 밸류에이션)
2. 해당 도메인의 표준 프레임워크 적용 (피칭 덱 구조, DCF, ARR/MRR 모델)
3. 초안 생성 → 사용자 검토 요청
4. 피드백 반영하여 최종본 완성

## 워크플로우

### 1단계: 목적 및 단계 확인
- 투자 단계 확인 (Pre-Seed / Seed / Series A / Series B+)
- 목표 산출물 선택 (IR 덱 / 재무 모델 / 투자자 업데이트 / 밸류에이션)
- 현재 비즈니스 지표 및 자료 수집

### 2단계: IR 피칭 덱 구성 (12슬라이드 표준)
1. **표지**: 회사명, 태그라인, 대표자
2. **문제**: 해결하는 고통점 (수치 포함)
3. **솔루션**: 핵심 제품/서비스
4. **시장 규모**: TAM/SAM/SOM
5. **비즈니스 모델**: 수익 구조
6. **트랙션**: 현재 성과 지표 (MAU, ARR, 고객 수)
7. **경쟁 분석**: 포지셔닝 매트릭스
8. **팀**: 핵심 멤버 경력
9. **로드맵**: 12~24개월 계획
10. **재무 전망**: 3년 P&L 요약
11. **투자 요청**: 금액, 사용 계획
12. **비전**: 장기 목표

### 3단계: 재무 모델 구성
- **매출 모델**: ARR/MRR, 성장률 가정, 채널별 분류
- **비용 구조**: 고정비 / 변동비, 인건비 계획
- **손익분석**: 월별 P&L, BEP(손익분기점) 시점
- **현금흐름**: Runway 계산, Burn Rate
- **밸류에이션**: ARR Multiple, DCF, 유사 기업 비교

### 4단계: 검토 및 정제
- 수치 일관성 검증 (가정 → 결과 연결 확인)
- 투자자 예상 질문 Q&A 준비
- 최종 문서 완성

## 사용 예시

**예시 1**: "시리즈 A 투자용 IR 덱 만들어줘. 우리 회사 ARR 5억, MAU 3만명이야"
→ 12슬라이드 구조 → 각 슬라이드 초안 → 검토 → 완성본

**예시 2**: "3년 재무 모델 만들어줘. 구독 SaaS 모델이고 현재 월 100명 신규 가입"
→ 성장 가정 설정 → MRR 모델 → P&L → Runway → 시나리오 비교

## 실행 규칙

1. 사용자 요청 수신 → 해당 하네스 판별
2. `references/{id}.md` 로드 → 가이드에 따라 실행
3. 복잡 재무 분석 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 결과물 생성 후 사용자 검토 요청

## 문제 해결

| 상황 | 대응 |
|------|------|
| 현재 매출/지표가 없는 경우 | 업계 벤치마크와 목표 기반으로 가정 설정. 가정임을 명시 |
| 재무 데이터 형식이 다양한 경우 | 제공된 형식 그대로 수용 후 표준화. 필요 시 추가 항목 요청 |
| 투자 단계와 목표가 불명확한 경우 | 현재 매출 규모와 사용 목적으로 적합한 단계 추정 후 확인 |
