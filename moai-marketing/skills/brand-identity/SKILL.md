# Brand Identity -- 브랜드 아이덴티티 설계 스킬

브랜드의 전략 수립부터 네이밍, 언어 아이덴티티, 비주얼 아이덴티티, 가이드라인 문서까지 체계적으로 설계한다.

## 지원 영역

- 신규 브랜드 아이덴티티 전체 설계
- 기존 브랜드 리브랜딩/리뉴얼
- 브랜드 네이밍 단독 개발
- 슬로건/태그라인 개발
- 톤앤매너 가이드 설계
- 비주얼 아이덴티티(색상, 타이포, 로고 컨셉) 설계
- 브랜드 가이드라인 문서 생성

## 트리거 키워드

브랜드 만들어줘, 브랜드 아이덴티티, 브랜드 네이밍, 이름 짓기, 슬로건, 태그라인,
톤앤매너, 브랜드 가이드라인, 로고 컨셉, CI 설계, 브랜드 전략, 브랜드 포지셔닝,
리브랜딩, 브랜드 리뉴얼, 기업 브랜딩, 브랜드 스토리, 브랜드 컬러, 브랜드 보이스

## 사용 예시

| 요청 | 실행 범위 |
|------|----------|
| "친환경 생활용품 브랜드 만들어줘. 2030 여성 타깃, 프리미엄" | 풀 파이프라인 (1~6단계 전체) |
| "펫 케어 스타트업 브랜드 네이밍해줘" | 1~3단계 (전략+네이밍) |
| "브랜드명은 '그리너리'로 정했어. 슬로건이랑 톤앤매너 만들어줘" | 4단계 (언어 아이덴티티) |
| "프랜차이즈 카페 브랜드 리뉴얼 해줘. 기존명 '모닝빈'" | 풀 파이프라인 (기존 요소 반영) |
| "테크 스타트업 비주얼 아이덴티티 설계해줘" | 2+5단계 (전략+비주얼) |
| "패션 브랜드 가이드라인 문서로 정리해줘" | 6단계 (가이드라인 생성) |

## 독립 실행 워크플로우

### 사전 수집: 브랜드 컨텍스트 체크리스트

작업 시작 전 다음 항목을 사용자에게 확인한다. 필수 항목이 누락되면 3가지 방향을 제안하여 선택을 유도한다.

| 항목 | 필수 | 설명 |
|------|-----|------|
| 사업 영역 | 필수 | 제품/서비스 종류, 업종 |
| 타깃 고객 | 권장 | 연령, 성별, 라이프스타일, 핵심 니즈 |
| 경쟁 브랜드 | 권장 | 벤치마크할 브랜드 2~3개 |
| 브랜드 방향 | 권장 | 원하는 이미지 키워드 (예: "프리미엄", "친근한") |
| 기존 확정 요소 | 선택 | 이미 정해진 브랜드명, 컬러, 로고 등 |
| 사업 규모 | 선택 | 스타트업/중소기업/대기업/프랜차이즈 |
| 타깃 시장 | 선택 | 국내 전용/글로벌 진출 계획 |

### 1단계: 시장 분석 및 경쟁사 벤치마킹

1. 해당 업종의 시장 현황을 분석한다
2. 경쟁 브랜드 3개 이상을 벤치마킹한다:
   - 브랜드명, 슬로건, 핵심 메시지
   - 사용 색상, 타이포그래피, 비주얼 톤
   - 포지셔닝 전략, 타깃 고객
3. 경쟁사 대비 차별화 기회를 도출한다
4. 산출물: `_workspace/01_brand_strategy.md`의 시장 분석 섹션

### 2단계: 브랜드 전략

1. **포지셔닝맵** 설계:
   - 2축 매트릭스 (예: 가격-품질, 전통-혁신)
   - 경쟁사 위치 표시 + 자사 목표 위치 설정
2. **타깃 페르소나** 개발:
   - 이름, 나이, 직업, 라이프스타일
   - 핵심 니즈, 페인포인트, 미디어 소비 패턴
3. **아키타입 선택** (`archetype-framework.md` 참조):
   - 브랜드 키워드에서 후보 2~3개 도출
   - 주 아키타입(70%) + 보조 아키타입(30%) 조합
   - 금기 조합 확인
4. **브랜드 에센스** 정의:
   - 미션 (존재 이유)
   - 비전 (미래 목표)
   - 핵심 가치 3~5개
5. 산출물: `_workspace/01_brand_strategy.md`

### 3단계: 네이밍

`naming-methodology.md` 레퍼런스를 반드시 참조한다.

1. **후보 생성** (최소 15개):
   - 12가지 네이밍 기법 중 3개 이상 활용
   - 한글명, 영문명, 한영 혼합명을 골고루 포함
2. **1차 스크리닝** (10개로 축소):
   - 발음 용이성, 기억 용이성 기준
   - 부정적 연상 제거
3. **SMILE 평가** (상위 5개 선정):
   - 5개 기준 x 1~5점 매트릭스 작성
   - 합격 기준: 총점 20/25 이상
4. **도메인/상표 사전검증** (상위 3개):
   - .com / .co.kr / .kr 도메인 가용성 확인 지침
   - KIPRIS 상표 검색 절차 안내
   - SNS 핸들(@username) 가용성 확인
   - 네이버/구글 검색 결과 확인
5. **최종 추천**: TOP 3를 순위와 함께 제시, 각각 선정 이유 명시
6. 산출물: `_workspace/02_naming_candidates.md`

### 4단계: 언어 아이덴티티

1. **슬로건 개발** (5개 이상):
   - 핵심 가치 기반, 7단어 이내
   - 한국어 슬로건 + 영문 슬로건 각각
   - 각 슬로건의 의도와 활용 맥락 설명
2. **태그라인** 개발:
   - 슬로건과 차별화된 서브 메시지
   - 캠페인별 변주 가능한 구조
3. **브랜드 스토리** 작성 (`archetype-framework.md`의 5막 구조 참조):
   - 창업 동기(Why) -> 발견(Insight) -> 해결(Solution) -> 증거(Proof) -> 초대(Invitation)
   - 300자/500자/1000자 3개 버전
4. **톤앤매너 가이드**:
   - 아키타입 기반 보이스 특성 3가지 + 예문
   - 안티보이스(금지 사항) 2가지 + 예문
   - 톤 스펙트럼 (격식-캐주얼, 유머-진지, 열정-절제, 존중-도전)
   - 채널별 톤 조정 (웹사이트/앱/SNS/이메일/고객센터)
5. 산출물: `_workspace/03_verbal_identity.md`

### 5단계: 비주얼 아이덴티티

`color-psychology.md` 레퍼런스를 반드시 참조한다.

1. **색상 시스템** 설계:
   - 브랜드 에센스에서 감정 키워드 추출
   - 경쟁사 색상 분석 및 차별화
   - 60-30-10 법칙 적용 (Primary/Secondary/Accent)
   - 색상 코드 명세 (HEX, RGB, CMYK, Pantone 참고값)
   - WCAG AA 이상 접근성 대비율 확보
   - 사용 금지 규칙 명시
2. **타이포그래피** 선정:
   - 한글 서체 + 영문 서체 페어링
   - 제목/본문/캡션별 서체 및 크기 규정
   - 무료 서체 우선, 유료 서체 시 대안 명시
3. **로고 컨셉** 제안 (2개 이상):
   - 각 컨셉의 디자인 방향 텍스트 설명
   - 워드마크/심볼마크/콤비네이션 중 권장 유형
   - 최소 크기, 여백 규정, 금지 사용 예시
4. 산출물: `_workspace/04_visual_identity.md`

### 6단계: 브랜드 가이드라인 문서 생성

`brand-guide-template.md` 템플릿을 기반으로 통합 문서를 생성한다.

1. 1~5단계 산출물을 통합한다
2. 채널별 적용 가이드를 추가한다 (웹/앱/SNS/인쇄물/명함)
3. 브랜드 용어집을 작성한다 (사용 용어 / 금지 용어)
4. 산출물: `_workspace/05_brand_guideline.md`

## 산출물 형식

모든 산출물은 `_workspace/` 디렉토리에 마크다운 형식으로 저장한다.

| 파일 | 내용 | 의존 |
|------|------|------|
| `01_brand_strategy.md` | 시장 분석, 포지셔닝맵, 페르소나, 아키타입, 에센스 | 없음 |
| `02_naming_candidates.md` | 네이밍 후보 15개+, SMILE 평가, TOP 3 추천 | 01 |
| `03_verbal_identity.md` | 슬로건, 태그라인, 스토리, 톤앤매너 | 01, 02 |
| `04_visual_identity.md` | 색상 시스템, 타이포, 로고 컨셉 | 01, 02 |
| `05_brand_guideline.md` | 통합 가이드라인, 채널별 적용, 용어집 | 01~04 |

## 핵심 규칙

1. 네이밍 후보는 반드시 15개 이상 생성하고 SMILE 매트릭스로 정량 평가한다
2. 아키타입은 주(70%) + 보조(30%) 조합으로 설계한다. 금기 조합을 반드시 확인한다
3. 슬로건은 한국어/영문 각 5개 이상 제시한다. 7단어 이내를 원칙으로 한다
4. 색상 시스템은 60-30-10 법칙을 따르며 WCAG AA 접근성을 확보한다
5. 모든 산출물은 사용자가 바로 활용할 수 있는 완성된 문서여야 한다
6. 실제 로고 파일(.ai, .svg) 제작, 인쇄물 시안, 상표 출원 대행은 이 스킬의 범위가 아니다
7. 도메인/상표 확인이 불가능한 경우 "[확인 필요]"로 표시하고 사용자에게 직접 확인을 권고한다

## 실행 규칙

1. 기존 확정 요소가 있으면 해당 단계를 건너뛰고 기존 정보를 `_workspace/`에 기록한다
2. 요청 범위에 따라 필요한 단계만 실행한다 (네이밍만, 슬로건만 등)
3. 레퍼런스 파일은 해당 단계 진입 시 Read 도구로 로드한다
4. 각 단계 완료 후 사용자에게 중간 결과를 보고하고, 다음 단계 진행 여부를 확인한다
5. 사용자 입력이 모호한 경우 3가지 방향을 제안하여 선택을 유도한다

## 이 스킬을 사용하지 말아야 할 때

| 요청 | 적합한 스킬 |
|------|-----------|
| 개인 브랜딩, 퍼스널 브랜딩 | `personal-branding` |
| 광고 캠페인 기획, 미디어 플래닝 | `campaign-planner` |
| SNS 콘텐츠 제작 | `sns-content` |
| 디자인 시스템/컴포넌트 구축 | `design-system` |

## 문제 해결

| 문제 | 대응 |
|------|------|
| 사업 영역이 불명확 | 3가지 사업 영역+포지셔닝 방향을 제안하여 선택 유도 |
| 네이밍 후보가 마음에 안 듦 | 사용자 피드백 수집 후 다른 네이밍 기법 3가지로 재생성 |
| 도메인/상표 확인 불가 | "[확인 필요]"로 표시, KIPRIS/도메인 조회 사이트 안내 |
| 경쟁사 정보 부족 | 일반 지식 기반으로 작업, "실시간 시장 데이터 미반영" 명시 |
| 비주얼 시안 요청 | 텍스트 컨셉+디자인 디렉션만 제공, 디자인 도구 활용 안내 |
