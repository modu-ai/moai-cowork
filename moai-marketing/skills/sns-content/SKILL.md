4. 피드백 반영하여 최종본 완성

## 트리거 키워드

SNS, 인스타그램, 네이버 블로그, 카카오, 콘텐츠, 브랜딩, 퍼스널 브랜딩, 브랜드 보이스

## 워크플로우

### 1단계: 목표 채널 및 콘텐츠 유형 확인
- 대상 채널 선택 (인스타그램 / 네이버 블로그 / 카카오 / 유튜브 / 전체)
- 콘텐츠 유형 선택 (피드 / 릴스 / 스토리 / 블로그 / 뉴스레터)
- 브랜드 톤 및 타겟 오디언스 확인

### 2단계: 채널별 최적화 작성
**인스타그램:**
- 캡션 (150자 이내 핵심 + 콜투액션)
- 해시태그 (15~25개, 대·중·소 믹스)
- 릴스: 후킹 첫 3초 + 스크립트 구성

**네이버 블로그 (C-Rank 최적화):**
- 제목 (핵심 키워드 포함, 25자 이내)
- 서론-본론-결론 구조, 2,000자 이상
- 경험 기반 콘텐츠 강조 (D.I.A. 기준)

**카카오 채널:**
- 짧고 명확한 메시지 (카카오톡 특성)
- 이모지·구분선 활용
- CTA 버튼 문구

### 3단계: 브랜드 전략 (선택)
- 브랜드 보이스 가이드: 톤, 키워드, 금지어
- 콘텐츠 달력: 주제별 월간 계획
- 퍼스널 브랜딩: 개인 포지셔닝 + 콘텐츠 방향성

### 4단계: 리퍼포징 (선택)
- 블로그 → 인스타 카드뉴스 → 유튜브 쇼츠 → 뉴스레터 변환

## 사용 예시

**예시 1**: "스킨케어 브랜드 인스타 피드 포스팅 5개 써줘. 타겟은 30대 직장 여성"
→ 타겟 분석 → 톤 설정 → 주제 5개 → 캡션 + 해시태그 세트

**예시 2**: "내 유튜브 강의 내용을 인스타·블로그·뉴스레터로 리퍼포징해줘"
→ 원본 분석 → 채널별 포맷 변환 → 각 채널 최적화 완성

## 실행 규칙

1. 사용자 요청 수신 → 하네스/실행 모듈 판별
2. 하네스 요청 → `references/{id}.md` 로드 → 전략 가이드 실행
3. 콘텐츠 요청 → `references/sns-content/guide.md` 로드 → 콘텐츠 생성
4. `--deepthink` 또는 복잡 작업 → `mcp__sequential-thinking__sequentialthinking` 호출
5. 결과물 생성 후 사용자 검토 요청

## 문제 해결

| 상황 | 대응 |
|------|------|
| 브랜드 톤/가이드가 없는 경우 | 업종·타겟·경쟁사 기반으로 톤 추천 후 확인 |
| 참고 자료가 없는 경우 | 업종 베스트 프랙티스 기반으로 초안 생성. 브랜드 자료 보완 요청 |
| 다수 채널 동시 요청 시 | 핵심 채널 1~2개 먼저 완성 후 리퍼포징으로 확장 |
