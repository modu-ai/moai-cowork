### 기본 실행 워크플로우 (references 미로드 시)

1. 사용자 요청에서 목표/산출물/채널/대상 추출
2. 해당 도메인의 표준 프레임워크 적용 (AIDA, AARRR, 고객 여정 5단계)
3. 초안 생성 → 사용자 검토 요청
4. 피드백 반영하여 최종본 완성

## 워크플로우

### 1단계: 캠페인 목표 및 유형 확인
- 목표 설정 (인지도 / 리드 / 전환 / 재구매)
- 채널 선택 (유료 광고 / SNS / 이메일 / 인플루언서)
- 예산 규모 및 기간 확인

### 2단계: 전략 수립
**캠페인 기획:**
- AARRR(Acquisition-Activation-Retention-Revenue-Referral) 퍼널 매핑
- 채널별 예산 배분 및 KPI 설정
- 크리에이티브 전략 방향

**A/B 테스트 설계:**
- 가설 수립 → 변수 정의 → 샘플 크기 계산
- 통계적 유의성 기준 설정 (p < 0.05)
- 결과 측정 지표 정의

**인플루언서 전략:**
- 메가/매크로/마이크로 인플루언서 티어 결정
- 선정 기준 (팔로워 수, 참여율, 타겟 부합도)
- 협찬 제안서 및 ROI 측정 방법

### 3단계: 이커머스 상세페이지 (선택)
- 제품 핵심 USP 추출
- 썸네일 → 상단 배너 → 제품 특징 → 사용 후기 → CTA 구성
- 쿠팡/스마트스토어 최적화 키워드 포함

### 4단계: 산출물 생성 및 검토

## 사용 예시

**예시 1**: "신제품 런칭 캠페인 기획해줘. 예산 500만원, 타겟은 2030 여성"
→ 채널 믹스 → 예산 배분 → KPI 설정 → 크리에이티브 방향 → 타임라인

**예시 2**: "스마트스토어 상세페이지 써줘. 제품은 천연 비타민C 세럼"
→ USP 추출 → 상세페이지 구조 → 카피 작성 → 키워드 최적화

## 실행 규칙

1. 사용자 요청 수신 → 하네스/실행 모듈 판별
2. 하네스 요청 → `references/{id}.md` 로드 → 전략 가이드 실행
3. 실행 모듈 요청 → 해당 `references/{module}/guide.md` 로드 → 콘텐츠/이미지 생성
4. `--deepthink` 또는 복잡 작업 → `mcp__sequential-thinking__sequentialthinking` 호출
5. 결과물 생성 후 사용자 검토 요청

## 문제 해결

| 상황 | 대응 |
|------|------|
| 예산이나 채널 정보가 없는 경우 | 업종·목표 기반 추천 예산 배분 제안 후 확인 |
| 제품 정보가 부족한 경우 | 핵심 USP와 타겟 고객 2~3가지 확인 후 진행 |
| 이미지/시각 자료가 필요한 경우 | AI 이미지 생성 모듈 또는 이미지 방향 텍스트 가이드 제공 |

## 이 스킬을 사용하지 말아야 할 때

- SNS 콘텐츠 단건 작성이나 브랜드 보이스 설정 → moai-marketing/sns-content 사용

- 마케팅 성과 분석 보고서 → moai-marketing/performance-report 사용
- 단순 정보 검색이나 일반 질문 → 일반 대화로 해결
