# 스케줄 생성 (Create Schedule)

> moai-schedules v1.0.0

## 개요

Claude Cowork의 `/schedule` 명령을 활용하여 반복 자동화 태스크를 설정합니다. 2026년 2월 25일 Cowork에 출시된 Scheduled Tasks 기능으로, 프롬프트를 한 번 작성하면 지정한 주기마다 Claude가 자동으로 실행합니다. 아침 브리핑, 주간 보고서, 월간 분석 등 반복 업무를 완전 자동화합니다.

## 트리거 키워드

스케줄 생성, 자동화 설정, 반복 작업, 매일 실행, 매주 자동, 스케줄 등록, 타이머, 자동 보고, cron, 예약 실행, /schedule

## Cowork Scheduled Tasks 기본 정보

### 출시 정보
- **출시일**: 2026년 2월 25일 (Claude 공식 X 계정 발표)
- **대상 플랜**: Pro, Max, Team, Enterprise (유료 플랜 전용)
- **실행 환경**: Claude Desktop 앱 + 컴퓨터 켜진 상태에서만 실행

### 지원 실행 주기

| 주기 | 설명 | 예시 |
|------|------|------|
| 매시간 | 1시간마다 실행 | 실시간 데이터 모니터링 |
| 매일 | 지정 시간에 실행 | 아침 8:30 브리핑 |
| 매주 | 요일 + 시간 지정 | 매주 월요일 9am 보고 |
| 평일만 | 월~금 실행 | 업무일 기준 작업 |
| 수동 실행 | On-demand | 필요 시 즉시 실행 |

### 중요 제한사항
- 컴퓨터가 절전 상태이거나 앱이 닫힌 경우 스케줄 건너뜀
- 건너뛴 작업은 앱 재시작 시 자동으로 실행되고 알림 발송
- 실행 기록은 태스크 히스토리에서 확인 가능

## 스케줄 생성 방법

### 방법 1: /schedule 명령 사용
```
/schedule
```
→ Claude가 스케줄 설정을 위한 대화 시작. 이름, 실행 주기, 프롬프트 내용 안내.

### 방법 2: Scheduled Tasks 페이지에서 직접 생성
1. 사이드바에서 "Scheduled" 클릭
2. "+ New task" 클릭
3. 다음 항목 입력:
   - **이름**: 스케줄 태스크 식별 이름
   - **설명**: 태스크 목적 (선택)
   - **프롬프트**: Claude에게 전달할 지시문
   - **실행 주기**: 매일/매주/수동 등
   - **모델**: 기본 Claude 또는 특정 모델 선택
   - **작업 폴더**: 파일 읽기/쓰기 대상 폴더 (선택)
4. "Save" 클릭

## 사전 정의 스케줄 템플릿

| 이름 | 대상 스킬 | 실행 주기 | 권장 시간 |

## 효과적인 스케줄 프롬프트 작성법

### 좋은 프롬프트 예시
```
오늘의 아침 브리핑을 작성해줘:
1. 오늘 날짜와 요일
2. 오늘 완료해야 할 상위 3개 우선순위 (context 폴더의 tasks.md 참조)
3. 이번 주 주요 마감일 정리
형식: 마크다운, 간결하게 500자 이내
```

### 프롬프트 작성 원칙
- 구체적인 출력 형식 명시 (마크다운/표/텍스트)
- 참조할 파일이 있으면 경로 명시 (작업 폴더 기준)
- 길이 제한 명시로 일관된 출력 유지
- 날짜/시간 컨텍스트는 Claude가 자동 인식

## 참고 사항

- **컴퓨터 절전 주의**: 야간 스케줄은 컴퓨터를 켜 두거나 절전 모드 해제 설정 필요
- **사용량 고려**: 스케줄 태스크는 일반 채팅 대비 사용량 소모 多. 복잡한 작업은 배치 처리 권장
- **Team/Enterprise**: 관리자가 Cowork 액세스 권한을 제어하므로 사전 확인 필요
- **파일 연동**: 작업 폴더 설정 시 로컬 파일 읽기/쓰기 자동화 가능 (보고서 파일 자동 저장 등)

## 문제 해결

| 상황 | 해결 방법 |
|------|-----------|
| 컴퓨터 절전 상태에서 실행 실패 | 스케줄 시간대를 활동 시간으로 변경하거나, macOS/Windows에서 절전 방지 설정을 적용하세요 |
| Cowork 플랜 제한 | Scheduled Tasks는 Pro/Max/Team/Enterprise 플랜 전용입니다. 플랜 업그레이드가 필요합니다 |
| 스케줄이 건너뜀 반복 | 실행 시간대에 앱이 열려 있는지 확인하세요. 자주 건너뛰면 실행 시간을 활동 시간으로 조정하세요 |
| 프롬프트 출력 불일치 | 출력 형식과 길이를 프롬프트에 명시적으로 작성해 주시면 도와드립니다 |

## 이 스킬을 사용하지 말아야 할 때

- **Cowork Scheduled Tasks 기능이 꺼져 있는 경우** → Claude Desktop 앱 Settings에서 Scheduled Tasks가 활성화되어 있는지 확인하세요
- **무료 플랜(Free) 사용 중** → Scheduled Tasks는 유료 플랜 전용입니다
- **스케줄 조회 또는 관리** → moai-schedules:list-schedules 또는 moai-schedules:manage-schedule 스킬을 사용하세요
- **서버 사이드 자동화 또는 API 기반 스케줄링** → cron job, GitHub Actions, AWS EventBridge 등 서버 도구를 사용하세요
