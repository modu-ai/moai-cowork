# 스케줄 관리 (Manage Schedule)

> moai-schedules v1.0.0

## 개요

Claude Cowork에 등록된 자동화 스케줄을 수정, 삭제, 일시정지, 재개합니다. 프롬프트 내용 업데이트, 실행 주기 변경, 작업 폴더 재지정 등 스케줄 전반의 유지 관리를 지원합니다. 스케줄 최적화 권고와 함께 사전 정의 템플릿 기반의 빠른 재설정도 제공합니다.

## 트리거 키워드

스케줄 수정, 스케줄 변경, 스케줄 삭제, 일시정지, 스케줄 재개, 자동화 끄기, 주기 변경, 프롬프트 업데이트, 스케줄 관리

## 지원 작업

| 작업 | 설명 |
|------|------|
| 수정 | 이름, 프롬프트, 주기, 작업 폴더 변경 |
| 일시정지 | 임시로 자동 실행 중단 (설정 유지) |
| 재개 | 일시정지된 스케줄 다시 활성화 |
| 삭제 | 스케줄 완전 제거 |
| 복제 | 기존 스케줄 복사 후 수정 |
| 즉시 실행 | 스케줄 주기와 무관하게 수동 실행 |

## 워크플로우

1. **대상 스케줄 선택**: list-schedules 스킬 연동 또는 직접 이름 입력
2. **작업 유형 선택**: 수정/일시정지/재개/삭제
3. **변경 내용 확인**:
   - 수정: 변경 전후 비교 표시
   - 삭제: 실행 이력 소실 경고
4. **Cowork UI 조작 가이드 제공**: 구체적인 클릭 경로 안내
5. **변경 후 검증**: 다음 실행 시간 재확인

## Cowork에서 스케줄 관리하는 방법

### 수정/삭제 경로
1. 사이드바 **"Scheduled"** 클릭
2. 대상 스케줄 선택
3. 오른쪽 상단 **"⋮" 메뉴** 또는 편집 아이콘 클릭
4. Edit / Pause / Delete 선택
5. 변경 사항 저장

### 즉시 실행 (수동 트리거)
- 스케줄 상세 페이지에서 **"Run now"** 버튼 클릭
- 주기와 무관하게 즉시 1회 실행
- 다음 정기 실행 일정에는 영향 없음

## 사전 정의 스케줄 템플릿 관리

기존 스케줄을 업데이트하거나 새로 설정할 때 참조하는 표준 템플릿:

### 매일 아침 브리핑
```
이름: 매일 아침 브리핑
주기: 매일 오전 8:30
연결 스킬: moai-business:daily-briefing
프롬프트:
  오늘의 아침 업무 브리핑을 작성해줘:
  1. 오늘 날짜 및 주요 일정
  2. 미완료 태스크 Top 3 (tasks.md 참조)
  3. 오늘의 우선순위 권고
  형식: 마크다운, 300자 이내
```

### 주간 마케팅 리포트
```
이름: 주간 마케팅 리포트
주기: 매주 월요일 오전 9:00
연결 스킬: moai-marketing:performance-report
프롬프트:
  지난 주 마케팅 성과를 요약해줘:
  - 채널별 KPI 달성 현황
  - 전주 대비 증감
  - 이번 주 핵심 액션 3가지
  데이터 소스: reports/ 폴더 최신 파일 참조
```

### 월간 세무 체크
```
이름: 월간 세무 체크
주기: 매월 20일 오전 10:00
연결 스킬: moai-finance:tax-helper
프롬프트:
  이번 달 세무 체크리스트를 실행해줘:
  - 이번 달 말 세금 신고 마감일 확인
  - 부가세/원천세/법인세 납부 일정
  - 준비 서류 체크리스트
```

### 분기 사업 리뷰
```
이름: 분기 사업 리뷰
주기: 분기말 금요일 (3/31, 6/30, 9/30, 12/31 전 금요일)
연결 스킬: moai-business:strategy-planner
프롬프트:
  이번 분기 사업 리뷰 초안을 작성해줘:
  - 분기 목표 달성 현황 (OKR)
  - 주요 성과 및 도전 과제
  - 다음 분기 전략 방향
```

### 주간 CS 요약
```
이름: 주간 CS 요약
주기: 매주 금요일 오후 5:00
연결 스킬: moai-support:escalation-manager
프롬프트:
  이번 주 고객 지원 현황을 요약해줘:
  - 총 수신 티켓 수 및 처리율
  - 에스컬레이션 건수
  - Top 3 반복 이슈
  - 다음 주 개선 권고
```

## 스케줄 최적화 권고

| 증상 | 원인 | 권고 조치 |
|------|------|-----------|
| 자주 건너뜀 | 컴퓨터 절전 시간대 | 실행 시간을 활동 시간대로 변경 |
| 출력 품질 저하 | 프롬프트 불명확 | 출력 형식 및 데이터 소스 명시 강화 |
| 사용량 한도 초과 | 스케줄 과다 | 유사 스케줄 통합 또는 주기 조정 |
| 불필요한 스케줄 | 목적 종료 | 일시정지 후 30일 내 재검토 또는 삭제 |

## 산출물

- 스케줄 수정/삭제/일시정지 가이드 (단계별 UI 안내)
- 변경 전후 비교 표
- 업데이트된 프롬프트 초안
- 스케줄 최적화 권고안

## 참고 사항

- **삭제 주의**: 스케줄 삭제 시 실행 이력도 함께 삭제됨. 이력 보존 필요 시 별도 로그 파일 생성 권장
- **일시정지 vs 삭제**: 향후 재사용 가능성 있으면 삭제 대신 일시정지 선택
- **주기 변경 효과**: 주기 변경 시 다음 실행 시간은 변경 직후부터 새 주기로 계산
- **Team 플랜**: 팀 관리자가 Cowork 스케줄 관리 권한을 제어할 수 있음. 조직 정책 확인 필요
