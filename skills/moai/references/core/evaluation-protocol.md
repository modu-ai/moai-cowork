# evaluation-protocol.md — 평가 프로토콜

## 개요
MoAI 하네스의 산출물과 프로세스를 체계적으로 평가하는 프로토콜입니다.
정확성, 완전성, 실용성, 톤 적합성, 현지화 적합성 5개 차원에서 평가합니다.

---

## 1. 평가 프레임워크

### 1-1. 5개 평가 차원

| 차원 | 정의 | 가중치 | 만점 |
|-----|------|-------|------|
| **정확성** | 사실/데이터의 정확도 | 20% | 100 |
| **완전성** | 요구사항 충족도 | 25% | 100 |
| **실용성** | 실행 가능성/유용성 | 25% | 100 |
| **톤 적합성** | 사용자 선호 톤 일치도 | 15% | 100 |
| **현지화** | 로케일/문화 적합도 | 15% | 100 |

**최종 점수 = Σ(차원_점수 × 가중치)**

---

## 2. 정확성 평가

### 2-1. 체크리스트

```
신뢰성 검증:
[ ] 인용된 통계/데이터의 출처 명확
[ ] 날짜/숫자의 정확성 (± 10% 오류 범위)
[ ] 용어 정의의 정확성
[ ] 상충하는 정보 없음
[ ] 최신 정보 (90일 이내)

채점 기준:
90-100: 모든 사실 검증됨
70-89: 1-2개 부정확 요소
50-69: 여러 부정확 요소 또는 오래된 정보
0-49: 심각한 오류 또는 신뢰성 부족
```

### 2-2. 자동 검증
```
AI 기반 검증:
- 사실 확인 (웹 검색)
- 논리 일관성 검사
- 시간 민감도 평가
- 출처 신뢰도 평가
```

## 3. 완전성 평가

### 3-1. 요구사항 체크리스트

```
사용자 요청 항목별 충족도:

요청: "블로그 글 작성 (SEO 최적화)"
[ ] 제목 (메인 키워드 포함)
[ ] 메타 디스크립션
[ ] 본문 (2,000자 이상)
[ ] 내부 링크 (3개 이상)
[ ] 외부 링크 (2개 이상)
[ ] 이미지 제안 (섹션별)
[ ] CTA (클릭 유도)

충족 항목 수 / 전체 항목 수 = 완전성 %
```

### 3-2. 깊이 평가
```
기본 완전성: 60점
  + 예시 추가: +10점
  + 시각화 제안: +10점
  + 대안 제시: +10점
  + 추가 참고자료: +10점
→ 최대 100점
```

## 4. 실용성 평가

### 4-1. 실행 가능성 점검

```
[ ] 단계별 설명이 명확한가?
[ ] 필수 도구/리소스 명시되었는가?
[ ] 소요 시간 예상치가 있는가?
[ ] 예상 결과가 명확한가?
[ ] 예외 케이스 처리가 있는가?
[ ] 성공/실패 기준이 명확한가?

점수:
90-100: 모두 충족 + 비용 추정치 있음
70-89: 대부분 충족
50-69: 절반 이상 충족
0-49: 부족하거나 불명확
```

### 4-2. 맥락 반영도
```
[ ] 사용자의 현 상황 파악했는가?
[ ] 업계/산업 특성 반영했는가?
[ ] 조직 규모 맞게 제안했는가?
[ ] 예산 수준 고려했는가?
[ ] 기술 수준 고려했는가?

점수:
90-100: 5개 모두 반영
70-89: 4개 반영
50-69: 3개 반영
0-49: 2개 이하
```

## 5. 톤 적합성 평가

### 5-1. 톤 요소별 평가

```
사용자 프로필 톤: "professional yet approachable"

요소별 체크:
[ ] 어휘 수준 적절 (너무 기술적이거나 너무 단순하지 않음)
[ ] 문장 길이 적절 (10~20자 평균)
[ ] 이모지/특수문자 사용 적절
[ ] 첫 인상 신뢰도 (타사 대비 경쟁력)
[ ] 호칭/경어 일관성
[ ] 유머/따뜻함 정도

채점:
90-100: 완벽하게 일치
70-89: 대체로 일치, 1-2개 요소 조정 필요
50-69: 부분 일치, 3-4개 요소 조정 필요
0-49: 거의 일치하지 않음
```

### 5-2. 일관성 검사
```
[ ] 문단별 톤이 일관적인가?
[ ] 용어 선택이 일관적인가? (예: "고객" vs "클라이언트")
[ ] 스타일이 회사 브랜드 톤과 맞는가?
[ ] 시간 경과에 따라 톤 변화 있는가?
```

## 6. 현지화 적합성 평가

### 6-1. 로케일별 체크

```
사용자 로케일: 한국 (ko_KR)

[ ] 언어: 한국어 문법 정확, 자연스러움
[ ] 날짜 형식: YYYY-MM-DD (ISO 8601)
[ ] 숫자 형식: 1,000,000 (쉼표)
[ ] 통화: KRW 사용
[ ] 시간대: 한국 시간 기준
[ ] 법규: 한국 규제 준수 (개인정보보호법, 전자상거래법)
[ ] 문화: 한국식 비즈니스 문화 반영
[ ] 예시: 한국 사례/회사명 사용

점수:
90-100: 8개 모두 충족
70-89: 6-7개 충족
50-69: 4-5개 충족
0-49: 3개 이하
```

### 6-2. 문화 민감도
```
[ ] 종교/정치적 편향 없음
[ ] 성별/나이 차별 없음
[ ] 문화적 존중심 있음
[ ] 현지 관습 존중
```

## 7. 등급 매핑

### 7-1. 최종 점수 → 등급

```
평가 점수 (가중평균) → 등급

S등급 (95-100): 탁월함
A등급 (85-94): 우수함
B등급 (75-84): 양호함
C등급 (65-74): 보통
D등급 (0-64): 미흡

예시 계산:
정확성: 92 × 0.20 = 18.4
완전성: 88 × 0.25 = 22.0
실용성: 85 × 0.25 = 21.25
톤적합성: 80 × 0.15 = 12.0
현지화: 87 × 0.15 = 13.05
────────────────────────
합계: 86.7 → A등급 (우수)
```

## 8. 자동 평가 시스템

### 8-1. 평가 엔진
```python
def auto_evaluate(output, harness, profile, user_feedback=None):
  dimensions = {
    'accuracy': evaluate_accuracy(output),
    'completeness': evaluate_completeness(output, harness),
    'practicality': evaluate_practicality(output),
    'tone_fit': evaluate_tone_fit(output, profile),
    'localization': evaluate_localization(output, profile)
  }
  
  weighted_score = sum(
    score * weights[dim] 
    for dim, score in dimensions.items()
  )
  
  # 사용자 피드백 반영
  IF user_feedback:
    weighted_score = adjust_with_feedback(weighted_score, user_feedback)
  
  grade = score_to_grade(weighted_score)
  return {
    'score': weighted_score,
    'grade': grade,
    'dimensions': dimensions,
    'improvements': generate_improvements(dimensions)
  }
```

## 11. 평가 피드백 루프

### 11-1. 사용자 평가 vs 자동 평가 비교
```
자동 평가 A등급 (86.7)
사용자 평가 8점/10점 (80)

차이: -6.7점

분석:
→ 자동 평가가 실용성을 과평가
→ 사용자는 구체적 실행 스텝 부족 지적
→ 규칙 업데이트 필요 (실용성 가중치 증대)
```

### 11-2. 학습 루프
```
매 평가마다:
1. 자동 평가 점수 기록
2. 사용자 피드백 수집
3. 차이점 분석
4. 규칙 조정 (필요시)
5. 다음 작업에 반영

→ evolution-protocol.md 참조
```
