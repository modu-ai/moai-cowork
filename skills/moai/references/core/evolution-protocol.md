# evolution-protocol.md — 자기학습 진화 프로토콜

## 개요
MoAI가 매 작업 후 자동으로 반성하고 학습하여 규칙을 진화시키는 프로토콜입니다.
Self-Refine 사이클을 통해 지속적으로 개선됩니다.

---

## 1. Self-Refine 사이클

### 1-1. 5단계 프로세스

**Step 1: 실행 후 반성 (Auto Reflection)**
```
작업 완료 직후 자동 실행:

1. 결과물 검토 (생성된 콘텐츠/자동화 결과)
2. 프로세스 평가 (시간, 리소스, 효율)
3. 예상 vs 실제 비교
4. 문제점 식별
```

**Step 2: 피드백 수집**
```
사용자 평가 요청:

"이 결과물이 얼마나 도움이 되셨나요?"
[1] ⭐⭐⭐⭐⭐ (10점) — 매우 만족
[2] ⭐⭐⭐⭐☆ (8점) — 만족
[3] ⭐⭐⭐☆☆ (6점) — 보통
[4] ⭐⭐☆☆☆ (4점) — 미흡
[5] ⭐☆☆☆☆ (2점) — 불만족
[Other] 의견 추가

IF feedback < 7:
  "어떤 부분을 개선했으면 좋을까요?"
  → 개선 의견 수집
```

**Step 3: 패턴 발견**
```
feedback + reflection에서 패턴 추출:

IF 같은 문제가 3회 이상 반복:
  → "심각한 문제" 신호
  → 규칙 업데이트 필요
  
IF 특정 하네스에서 일관된 성공:
  → "좋은 패턴" 신호
  → 규칙 강화

IF 특정 사용자 그룹에서만 문제:
  → "세그먼트별 문제" 신호
  → 세부 규칙 추가
```

**Step 4: 규칙 업데이트**
```
발견된 패턴에 따라 규칙 개선:

IF 문제_유형 == "톤과 맥락 불일치":
  update_rule("01-{harness}.md", "톤 가이드" 상세화)
  
IF 문제_유형 == "누락된 정보":
  update_rule("context-collector.md", "B등급 추가")
  
IF 문제_유형 == "지역 규제 무지":
  update_rule("02-locale-*.md", "규제 내용 추가")

저장 위치: .moai/evolution/
├── self-refine-log.md (매 사이클 기록)
├── rule-updates.md (규칙 변경 이력)
└── pattern-database.md (패턴 발견 기록)
```

**Step 5: 교차 도메인 학습**
```
한 하네스에서의 학습을 다른 하네스에 적용:

Example:
content_generator에서 "사용자는 짧은 문단 선호" 발견
→ email_harness에도 동일 규칙 적용
→ automation_harness 결과 설명도 간결화

학습 전파:
00-moai-core.md (기본) → 모든 하네스에 적용
→ 개별 하네스 (01-*.md)에 특화된 내용 추가
```

## 2. 진화 로그 (.moai/evolution/)

### 2-1. self-refine-log.md

```markdown
# self-refine-log.md — 반성 로그

## 2026-04-04 (Day 1)

### 작업 1: content_generator — 블로그 글 작성
- **결과물**: "디지털 마케팅 트렌드 2026"
- **소요 시간**: 12분
- **사용자 평가**: 8점 ⭐⭐⭐⭐☆
- **사용자 의견**: "좋은데, 더 실행 가능한 액션 아이템이 필요해요"
- **자동 평가**: 완성도 85%, 실용성 75%
- **문제점**:
  1. 전략적 조언은 있지만 "실행 스텝"이 구체적이지 않음
  2. 내부 사례 부재 (귀사 사례 추가 필요)
- **패턴**: content_generator에서 "실행성" 약함

### 작업 2: automation_harness — 이메일 발송 자동화
- **결과물**: Zapier + Gmail 자동화 설정안
- **소요 시간**: 8분
- **사용자 평가**: 9점 ⭐⭐⭐⭐⭐
- **사용자 의견**: "정확해요. 바로 구현 가능합니다."
- **자동 평가**: 완성도 90%, 실용성 95%
- **장점**:
  1. 단계별 설명이 명확
  2. 도구 선택 근거가 명시
  3. 비용 예상치 제공
- **패턴**: automation_harness에서 "구체성" 강함

## 진화 전략
→ content_generator에 "실행 스텝" 템플릿 추가
→ automation_harness의 "구체성" 방식을 content_generator에 적용
```

### 2-2. rule-updates.md

```markdown
# rule-updates.md — 규칙 변경 이력

## 2026-04-05 (Update 1)

### 변경 파일: .claude/rules/01-content_generator.md
- **변경 이유**: 사용자 피드백 "실행 아이템 필요"
- **변경 전**: "솔루션 3가지 제시"
- **변경 후**: "솔루션 3가지 + 각 솔루션별 실행 스텝 3-5개"
- **예상 효과**: 실용성 점수 +15%
- **롤백**: 필요시 revert 커밋 ID = abc123def456

### 변경 파일: .claude/rules/00-moai-core.md
- **변경 이유**: 블로그 vs 실행서 구분 필요
- **추가 항목**: "산출물 타입에 따른 상세도 가이드"
  - 블로그: 전략 + 배경지식
  - 실행서: 단계 + 도구 + 예상 시간
- **예상 효과**: 모든 콘텐츠 실용성 +10%
```

### 2-3. pattern-database.md

```markdown
# pattern-database.md — 패턴 데이터베이스

## Good Patterns (강화할 패턴)

### Pattern 1: 구체적 실행 스텝
```
조건: 자동화/프로세스 설명
성공률: 95% (평가 점수 > 8)

템플릿:
1. 목표 명시
2. 필수 도구 3가지 (각각 2-3줄 설명)
3. 단계별 실행 (5-7단계, 각 단계마다 예상 소요 시간)
4. 비용 추정 (초기/월간)
5. 예상 효과 (정성/정량)
```

### Pattern 2: 타사 사례 + 귀사 맥락
```
조건: 전략/계획 설명
성공률: 85% (평가 점수 > 7)

템플릿:
1. 개념 설명 (3-5줄)
2. 타사 사례 2가지 (결과물 중심)
3. 귀사 적용 가능성 분석
4. 귀사 현황 반영한 맞춤 제안
5. 리스크 및 대안
```

## Bad Patterns (제거할 패턴)

### Anti-Pattern 1: 과도하게 이론적
```
증상: 사용자 피드백 "너무 학술적"
빈도: 5회/월
원인: context에 "전략적 깊이" 강조
조치: "실행성 가중치" 우선순위 상향
```

## Contextual Patterns (조건부 패턴)

### Pattern: 긴급도별 상세도
```
긴급도 높음 (< 24시간) → 간결한 액션 아이템만
긴급도 중간 (1주) → 전략 + 액션
긴급도 낮음 (> 1주) → 깊이 있는 분석

## 3. 롤백 메커니즘

### 3-1. 롤백 조건
```
IF 규칙_변경_후_평가_점수 < 규칙_변경_전_평가_점수 - 2점:
  → 자동 롤백 경고
  → 사용자 확인 후 롤백 실행
```

### 3-2. 롤백 프로세스
```bash
# 이전 규칙 버전으로 복원
/moai evolution --rollback --file=01-content_generator.md --version=v1.2

# 또는 특정 커밋으로 복원
/moai evolution --rollback --commit=abc123def456
```

### 3-3. 롤백 이력
```
.moai/evolution/rollback-log.md:

2026-04-10 10:30 Rollback 1
- 파일: 01-content_generator.md
- 이유: "실행 스텝" 추가 후 형식 오류
- 복원 버전: v1.2
- 담당: Auto-Rollback
```

## 4. 학습 속도 제어

### 4-1. 극복보기(Forgetting Curve) 관리
```
규칙 변경 후 모니터링:
- Day 1: 즉시 모니터링 (변경 적용 여부 확인)
- Day 7: 주간 평가 (지속 효과 확인)
- Day 30: 월간 평가 (안정성 확인)

IF Day 30 평가 < Day 7 평가:
  → 규칙이 부자연스러움
  → 수정 또는 롤백
```

### 4-2. 과도한 학습 방지
```
동시 진행 규칙 변경: 최대 3개
변경 간격: 최소 7일 (안정화 기간)
변경 규모: 최대 파일당 2-3개 섹션

→ 변수가 너무 많아서 원인 파악 불가 방지
```

## 6. 사용자 개입

### 6-1. 명시적 피드백
```
사용자 피드백 우선순위:
1. 명시적 의견 (예: "더 실용적으로")
2. 평점 데이터
3. 자동 감지 (문법 오류, 시간 초과)
```

### 6-2. 규칙 제안
```
/moai evolution --suggest

자동 제안 예:
"Rule 01-content_generator.md의 '실행성' 강화를 제안합니다.
최근 5개 작업 평가 점수 平均 8.2점이지만,
피드백 키워드 '실행 아이템'이 3회 반복되었습니다."

[1] 승인
[2] 거부
[3] 수정 후 승인
```
