## 워크플로우

### 1단계: 요청 유형 확인
- 계약서 검토 vs 법률 문서 신규 작성 구분
- 문서 유형 식별 (용역계약 / NDA / 파트너십 / 이용약관 / SLA 등)
- 한국법 또는 외국법 적용 여부 확인

### 2단계: 계약서 검토 (10대 리스크 체크)
- [ ] 계약 당사자 정보 정확성
- [ ] 계약 목적 및 범위 명확성
- [ ] 대금·지급 조건 및 지연 이자
- [ ] 납기·이행 조건 및 위약 조항
- [ ] 지적재산권 귀속 (특허, 저작권, 영업비밀)
- [ ] 비밀유지 조항 (범위, 기간, 예외)
- [ ] 손해배상 범위 및 한도 (직접 손해 vs 간접 손해)
- [ ] 계약 해지 조건 및 효과
- [ ] 분쟁 해결 (관할 법원, 준거법, 중재)
- [ ] 불공정 조항 (약관법 위반 가능성)

### 3단계: 법률 문서 작성 (선택)
**이용약관:**
- 전자상거래법·소비자보호법 준수
- 필수 포함 사항: 서비스 범위, 이용 제한, 책임 한계, 분쟁 해결

**개인정보처리방침:**
- 개인정보보호법 (2026.9.11. 개정) 반영
- 수집 항목, 이용 목적, 보유 기간, 제3자 제공, 파기 방법

**SLA:**
- 가용성 목표 (예: 99.9% 업타임)
- 장애 대응 시간 (P1/P2/P3 등급별)
- 위반 시 보상 방법

### 4단계: 수정 권고안 제시
- 리스크 조항 표시 및 대안 문구 제안
- 협상 포인트 우선순위 제시

## 사용 예시

**예시 1**: "IT 용역 계약서 검토해줘. 갑이 우리야. [계약서 내용]"
→ 10대 리스크 체크 → 불리한 조항 표시 → 수정 권고안 → 협상 포인트

**예시 2**: "SaaS 서비스 이용약관이랑 개인정보처리방침 만들어줘"
→ 서비스 범위 확인 → 이용약관 초안 → 개인정보처리방침 초안 → 검토

## 실행 규칙

1. 사용자 요청 수신 → 하네스/실행 모듈 판별
2. 법률 문서 요청 → `references/service-legal-docs.md` 로드 → 문서 작성
3. 계약서 검토 요청 → `references/contract/guide.md` 로드 → 한국법 기반 분석
4. **법률/규제 판단은 항상** `mcp__sequential-thinking__sequentialthinking` 호출
5. 결과물 생성 후 사용자 검토 요청

⚠️ 면책 고지: MoAI는 AI 어시스턴트이며 공인 법률 자문을 대체하지 않습니다. 중요한 법률 사안은 반드시 전문 법률가와 상담하세요.

## MCP 도구 활용 (korean-law)

korean-law MCP 서버가 연결되어 있으면 법제처 14개 도구를 활용합니다:
- **법령 검색**: 법령명 또는 키워드로 관련 조항 조회
- **조항 상세**: 특정 법률의 조문 전체 텍스트 확인
- **판례 조회**: 관련 판례 검색으로 해석 근거 확보
- **활용 순서**: 계약 유형 파악 → 관련 법령 검색 → 조항 대비 검토 → 판례 기반 리스크 평가

MCP 미연결 시: 내장된 법률 지식으로 일반 검토 수행. 최신 개정사항은 반영되지 않을 수 있음을 명시.

## 문제 해결

| 상황 | 대응 |
|------|------|
| 계약서 원문이 없는 경우 | 계약 유형과 주요 조건을 확인하여 표준 계약서 초안 제공 |
| 외국법 준거 계약인 경우 | 한국법 관점의 일반 검토 제공. 해당 국가 법률 전문가 검토 권고 |
| 전문 법률 용어가 불명확한 경우 | 쉬운 한국어로 해설 후 법률적 시사점 설명 |
| korean-law MCP 연결 실패 | 내장 법률 지식으로 진행. "최신 법령 확인 필요" 면책 표시 |
